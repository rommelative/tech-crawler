# Render.com 部署指南

## 部署前准备

你需要：
1. 一个 **GitHub** 账号
2. 安装 **Git** (https://git-scm.com/download/win)

## 部署步骤

### 第一步：创建 GitHub 仓库

1. 打开 https://github.com/new
2. 仓库名称填写：`tech-crawler`
3. 选择 **Public**
4. 点击 "Create repository"

### 第二步：初始化本地 Git

在 `tech-crawler-share` 文件夹中，右键选择 "Git Bash Here"，然后执行：

```bash
git init
git config user.name "你的名字"
git config user.email "你的邮箱"

# 添加所有文件
git add .
git commit -m "Initial commit"

# 关联 GitHub 仓库（把下面改成你创建的仓库地址）
git remote add origin https://github.com/你的用户名/tech-crawler.git

# 推送到 GitHub
git push -u origin main
```

### 第三步：部署到 Render

1. 打开 https://dashboard.render.com
2. 用 GitHub 账号登录
3. 点击 "New +" → "Web Service"
4. 选择你刚才创建的 `tech-crawler` 仓库
5. 配置如下：
   - **Name**: tech-crawler
   - **Environment**: Python
   - **Build Command**: pip install -r requirements.txt
   - **Start Command**: gunicorn app:app
6. 点击 "Create Web Service"

### 第四步：等待部署

- 部署大约需要 2-3 分钟
- 完成后会显示一个 URL，例如：`https://tech-crawler.onrender.com`
- 打开即可访问！

## 注意事项

1. **免费版限制**：Render 免费版会：
   - 15分钟无访问自动休眠
   - 每月750小时免费时长
   - 休眠后首次访问需要等待约30秒启动

2. **数据存储**：云端的收录数据保存在 `/tmp/collected.txt`，重启后会丢失。如需持久化，可以连接数据库。

## 文件说明

部署已包含以下文件：
- `requirements.txt` - Python 依赖
- `runtime.txt` - Python 版本
- `Procfile` - Render 启动配置
- `run.py` - 应用入口
